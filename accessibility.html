Web Accessibility

Learn how to create inclusive websites that work for everyone, including people with disabilities. Web accessibility ensures equal access to information and functionality.

Why Accessibility Matters

  • Inclusive Design
    Everyone deserves equal access to information
  • Legal Requirements
    Many countries have accessibility laws (ADA, WCAG)
  • Better SEO
    Accessible sites rank better in search engines
  • Wider Audience
    Reach more users including those with disabilities
  • Future-Proof
    Works with assistive technologies and new devices

WCAG Guidelines

Web Content Accessibility Guidelines (WCAG) provide the international standard for web accessibility.

Four Principles (POUR)

  • Perceivable
    Information must be presentable to users in ways they can perceive
  • Operable
    User interface components must be operable by all users
  • Understandable
    Information and operation must be understandable
  • Robust
    Content must be robust enough to work with current and future technologies

Semantic HTML

Use semantic HTML elements to provide meaning and structure to your content.

<!-- Good: Semantic HTML -->
<header>
    <nav>
        <ul>
            <li><a href="#home">Home</a></li>
        </ul>
    </nav>
</header>

<main>
    <article>
        <h1>Article Title</h1>
        <p>Article content...</p>
    </article>
</main>

<footer>
    <p>© 2024 Company</p>
</footer>

Alternative Text for Images

Always provide descriptive alt text for images.

<!-- Good alt text -->
<img src="puppy.jpg" alt="Golden retriever puppy playing in grass">

<!-- Bad alt text -->
<img src="puppy.jpg" alt="puppy">
<img src="puppy.jpg" alt=""> <!-- Empty alt for decorative images -->

<!-- Decorative images -->
<img src="divider.png" alt="" role="presentation">

Color and Contrast

Ensure sufficient color contrast for readability.

WCAG Contrast Requirements

  • Normal text: 4.5:1 contrast ratio
  • Large text (18pt+): 3:1 contrast ratio
  • Graphics and UI: 3:1 contrast ratio
/* Good contrast */
.text-primary {
    color: #000000;
    background: #ffffff;
}

/* Poor contrast - avoid */
.text-bad {
    color: #777777;
    background: #999999;
}

Keyboard Navigation

Ensure all interactive elements are accessible via keyboard.

/* Focus styles for keyboard navigation */
button:focus,
a:focus,
input:focus,
select:focus,
textarea:focus {
    outline: 2px solid #007bff;
    outline-offset: 2px;
}

/* Skip links for screen readers */
.skip-link {
    position: absolute;
    top: -40px;
    left: 6px;
    background: #000;
    color: #fff;
    padding: 8px;
    text-decoration: none;
}

.skip-link:focus {
    top: 6px;
}

Forms and Labels

Properly label form elements for screen readers.

<!-- Good: Explicit labels -->
<label for="email">Email Address</label>
<input type="email" id="email" name="email">

<!-- Good: Implicit labels -->
<label>
    Email Address
    <input type="email" name="email">
</label>

<!-- Bad: No labels -->
<input type="email" placeholder="Email Address">

Media Accessibility

  • Video
    Provide captions, transcripts, and audio descriptions
  • Audio
    Include transcripts and captions for podcasts/videos
  • Animations
    Respect user's motion preferences (prefers-reduced-motion)
/* Respect user motion preferences */
@media (prefers-reduced-motion: reduce) {
    *, *::before, *::after {
        animation-duration: 0.01ms !important;
        animation-iteration-count: 1 !important;
        transition-duration: 0.01ms !important;
    }
}

Testing Accessibility

  • Automated Tools
    WAVE, axe, Lighthouse accessibility audits
  • Manual Testing
    Keyboard navigation, screen reader testing
  • Color Contrast Checkers
    Online tools to verify contrast ratios
  • User Testing
    Test with users who have disabilities

Common Accessibility Mistakes

  • Missing alt text on images
  • Poor color contrast
  • Missing form labels
  • Non-keyboard accessible elements
  • Missing focus indicators
  • Auto-playing media without controls
  • Complex navigation without skip links
